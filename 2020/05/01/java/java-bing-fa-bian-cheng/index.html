  1. 使用synchronized的三种方法。
  • synchronized保护的是对象而不是代码，只要访问的是同一个对象的synchronized方法，即使是不同的代码，也会被按照同步顺序执行。
public class Test1 implements Runnable{

    private int num = 0;
    private Object o = new Object();

    public void m()
    {
        //synchronized锁定的是一个对象不是代码块
        synchronized (o)
        {
            num++;
            System.out.println(Thread.currentThread().getName()+"num =" + num);
        }
    }
    public static void main(String[] args) {
        Test1 test = new Test1();

        //10000各线程都访问同一个test对象
        for(int i = 0;i<10000;i++)
        {
            new Thread(test,"thread"+i).start();
        }
    } 
    
}

  • synchronized不能保证非同步的方法被同时执行。所以一般在保护变量的时候需要将所有的变量方法都加上synchronized。只有加了synchronized的方法在运行的时候才需要申请锁，其他方法不需要申请锁。
package com.c_001;

public class test2 implements Runnable{
    private int count = 10000;

    @Override
    public synchronized void run() {
        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        count --;
        System.out.println(Thread.currentThread().getName()+" count = "+count);
    }

    public void m2()
    {
        while(count>0)
        {
            count--;
            System.out.println(Thread.currentThread().getName()+" count = "+count);
        }
    }

    public static void main(String[] args) {
        test2 test = new test2();

        //10000各线程都访问同一个test对象
        for(int i = 0;i<10000;i++)
        {
            new Thread(()->test.run(),"thread"+i).start();
        }
        new Thread(()->test.m2(),"main1").start();
    }
}
  • 如果只对业务的写加锁，而不对读加锁的话就会产生脏读的问题。如果业务运行脏读，可以不加锁，脏读就是可能读到了没有写完的数据。
/**
 * 对业务的写加锁
 * 不对业务的读加锁
 * 容易产生脏读的现象
 */
public class test {
    String name;
    double money;


    public synchronized void write(String name,double money)
    {
        this.name = name;

        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }

        this.money = money;
    }

    public double read(String name)
    {
        return this.money;
    }

    public static void main(String[] args) {
        test test1 = new test();
        new Thread(()->test1.write("zhangsan",100)).start();
        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        System.out.println(test1.read("zhangsan"));
        try {
            Thread.sleep(500);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        System.out.println(test1.read("zhangsan"));
    }
}

  • atomicXXX多个方法之间并不构成原子性，只是单个的方法才有原子性。
  • 锁的粒度越细越好。
  • 锁定某个对象的时候如果该对象的属性发生改变，不会隐形锁的使用，但是如果o编程另一个对象，锁定的独享发生改变。所以应该避免将锁定对象的引用变成另个对象。
  • 不要以字符串常量作为锁定的对象，因为字符串常量常量区，可能两个引用指向的是同一个对象，锁定的也是同一个对象，容易给人锁定的是不同对象的误解。

  • 一个固定容量同步容器，拥有put和get方法，以及getCount方法，能够支持2个生产者线程以及10个消费者线程的阻塞调用,两种方法，一种使用sychronnized和wait实现，另一个中用condition实现。
package c_004;

import java.util.LinkedList;
import java.util.List;

public class Solution1<T> {
    final private List list= new LinkedList();
    final private int MAX = 10;
    private int count = 0;

    public synchronized void put(T t)
    {
        while(list.size()==MAX)
        {
            try {
                this.wait();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
        list.add(t);
        ++count;
        this.notifyAll();
    }

    public synchronized T get()
    {
        while(list.size()==0)
        {
            try {
                this.wait();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
        //list.get(count);
        T t = (T)list.remove(count-1);
        count--;
        this.notifyAll();
        return t;
    }

    public synchronized int getCount(){
        return count;
    }

    public static void main(String[] args) {
        Solution1<Integer> s1 = new Solution1<>();

        Integer total = 0;

        for(int i = 0;i<10;i++)
        {
            new Thread(()->{
                while(true)
                {
                    int ss = s1.get();
                    System.out.println(Thread.currentThread().getName()+ "get " + s1.getCount() + "is" + ss);
                }
            },"producer"+i).start();
        }

        for(int i = 0;i<2;i++)
        {
            new Thread(()->{
                while(true){
                    s1.put(s1.getCount());
                    System.out.println(Thread.currentThread().getName()+"set"+(s1.getCount())+"is" + s1.getCount());
                }
            },"consumer"+i).start();
        }
    }

}

import java.util.concurrent.locks.Condition;
import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;

public class Solution2<T> {
    private final List<T> list = new LinkedList<T>();
    private final int  MAX = 10;
    Lock lock = new ReentrantLock();
    Condition consumer = lock.newCondition();
    Condition producer = lock.newCondition();

    int count = 0;

    public void put(T t)
    {
        try {
            lock.lock();
            while(count>MAX)
            {
                producer.await();
            }
            list.add(t);
            count++;
            consumer.signalAll();
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            lock.unlock();
        }
    }

    public T get()
    {
        T t = null;
        try {
            lock.lock();
            while(count==0) {
                consumer.await();
            }
            t = list.remove(count-1);
            count--;
            producer.signalAll();
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            lock.unlock();
            return t;
        }
    }

    public int getCount(){
        return count;
    }

    public static void main(String[] args) {
        Solution2<Integer> s2 = new Solution2();

        for(int i = 0;i<10;i++)
        {
            new Thread(()->{
                while(true)
                {
                    int ss = s2.get();
                    System.out.println(Thread.currentThread().getName()+ "get " + s2.getCount() + "is" + ss);
                }
            },"producer"+i).start();
        }

        for(int i = 0;i<2;i++)
        {
            new Thread(()->{
                while(true){
                    s2.put(s2.getCount());
                    System.out.println(Thread.currentThread().getName()+"set"+(s2.getCount())+"is" + s2.getCount());
                }
            },"consumer"+i).start();
        }
    }
}

java并发队列

  • 无锁非阻塞并发队列ConcurrentLinkedQueue和ConcurrentLinkedDeque

  • 普通阻塞队列ArrayBlockingQueue和LinkedBlockingQueue和LinkedBlockingDeque

  • 优先级阻塞队列PriorityQueue

  • 延时阻塞队列DelayQueue

  • 其他阻塞队列SychronousQueue和LinkedTransferQueue

  • 无锁非阻塞是指这些队列不使用锁，所有的操作都可以立即执行，主要通过cas实现并发安全。

  • 阻塞队列是指这些队列使用锁和条件。很多操作需要先获得锁或者满足条件。

  • concurrentLinkedQueue和concurrentlinkedDeque都是基于链表是实现的，都没有大小的限制，也就是无界的，基于循环cas实现。

  • 阻塞队列的方法：

    //入队，如果队列满，等待直到队列有空间
void put(E e) throws InterrupteredException;
//出对，如果队列空，等待直到队列不空，返回元素的头部
void take(E e) throws InterrupteredException;
//入队，如果队列满，最多等待一段时间，如果超时返回false
boolean offer(E e,long timeout,TimeUnit unit) throws InterruptedException;
//出队，如果队列空，最多等待一段时间，如果超时返回null
E poll(long timeout,TimeUnit unit) throws InterruptedException;

  • ArrayBlockingQueue基于循环数组实现，有界，创建指定大小，
    LinkedBlockingQueue基于单向链表实现，可以指定有界，默认无界，
    和LinkedBlockingDeque和上面一个一样，但是是双向链表

    • 内部都是通过reentrantlock和条件是实现的。

  • priorityQueue大小没有限制无界，堆实现，使用一个锁reetrantlock保护所有访问，使用了一个条件协调阻塞等待。

  • delayQueue特殊的优先级队列，无界的，只有元素的延时过期的时候才能从队列中拿出来元素。

  • LinkedTransferQueue可以等待消费者接受后再返回。synchronousQueue没有存储元素，也就是他的入队操作需要等待另一个线程的出队操作。

线程池

基本概念和原理

  • Executor表示最简单的执行任务，是一个接口，改接口只有一个excute方法,该方法执行一个runnable任务，没有返回值，接口没有限定任务如何执行，你可以创建一个新的线程执行，也可以直接执行，或者是复用线程池的某个线程。

    package com.c_006;
import java.util.concurrent.Executor;
public class MyExcutor implements Executor{
    public static void main(String[] args) {
        new MyExcutor().execute(()-> System.out.println(11));
    }
    @Override
    public void execute(Runnable command) {
        new Thread(command).run();
        command.run();
    }
}

  • ExecutorService扩展了Executor，接口，定义了更多的服务，可以运行runable和callable方法，具体怎么运行不知道，基本方法有：

    public interface ExecutorService impelements Executor{
    <T> Future<T> summit(Callable<T> task);
    <T> Future<T> summit(Runnable<T> task,T result);
}

  • callable和runnable都是被其他线程调用的，但是callable有返回值，而runable没有。

  • Executors是操作executor的工具类。理解为工厂方法。

package com.c_006;
